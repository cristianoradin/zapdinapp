1. O que é o ZapDin PDV

O ZapDin PDV é um serviço instalado localmente na máquina do cliente (computador do caixa / PDV). Ele expõe uma API REST simples que o ERP chama para enviar mensagens WhatsApp — tudo sem depender de internet ou servidores externos no momento do envio.

✅ Tudo local: O ERP chama http://localhost:4600. O WhatsApp conecta nesta mesma máquina. Sem latência externa, sem dependência de internet no envio.

💡 Porta padrão: 4600. Se precisar alterar, mude PDV_PORT no .env e atualize a URL no ERP.
3. Configuração do PDV (.env)

O arquivo .env fica em C:\ZapDinPDV\.env. Os valores relevantes para o ERP são:

PDV_API_KEY
Chave secreta que o ERP deve enviar no header X-PDV-Key. Defina um valor seguro.
PDV_PORT
Porta onde o PDV escuta. Padrão: 4600. O ERP chama http://localhost:{PDV_PORT}.
PDV_NOME
Nome identificador deste PDV (ex: Caixa 01). Aparece nos logs e no status.
EVOLUTION_API_KEY
Chave interna do Evolution API. Padrão: zapdin-pdv-local. Não alterar sem necessidade.
Exemplo de .env
Arquivo: C:\ZapDinPDV\.env
# ZapDin PDV — Configuração EVOLUTION_URL=http://localhost:8080 EVOLUTION_API_KEY=zapdin-pdv-local ZAPDIN_URL=https://app.seuservidor.com.br ZAPDIN_PDV_TOKEN=pdv_xxxxxxxxxxxxxxxxxxx PDV_PORT=4600 PDV_API_KEY=minha-chave-secreta-123 # ← ERP usa esta chave PDV_NOME=Caixa 01
4. Autenticação

Todas as requisições devem incluir o header X-PDV-Key com o valor configurado em PDV_API_KEY no .env do PDV.

Header obrigatório em todas as requisições
X-PDV-Key: minha-chave-secreta-123 Content-Type: application/json
❌ Sem o header o PDV retorna 401 Unauthorized e a mensagem não é enviada.
5. Verificar Status do PDV

Antes de enviar, o ERP pode verificar se o PDV está ativo e o WhatsApp conectado.

GET /status Retorna estado do PDV e das sessões WhatsApp
Resposta de sucesso (200)
{ "ok": true, "pdv": "Caixa 01", "sessoes_conectadas": 1, ← checar se > 0 antes de enviar "sessoes": [{ "id": "a1b2c3", "status": "connected", ← connected | connecting | disconnected "phone": "5544999990000" }] }
⚠️ Se sessoes_conectadas for 0, o WhatsApp não está conectado. Oriente o operador a escanear o QR code em http://localhost:4600/setup/qr.
6. Enviar Comprovante de Venda
POST /erp/venda Envia comprovante de venda via WhatsApp
CampoReq.TipoDescrição
phoneSimstringNúmero do cliente. Ex: "44999990000" ou "+5544999990000"
numero_vendaSimstringNúmero ou código da venda. Ex: "001234"
nome_clienteSimstringNome completo do cliente
valor_totalSimstringValor total. Ex: "350,00"
forma_pagamentoNãostringEx: "PIX", "Cartão de Débito"
dataNãostringData da venda. Ex: "12/05/2026"
itensNãoarrayLista de produtos (ver estrutura abaixo)
observacaoNãostringObservação livre adicionada à mensagem
arquivo_base64NãostringPDF do comprovante em base64 — enviado após o texto
arquivo_nomeNãostringNome do arquivo. Ex: "comprovante_001234.pdf"
sessao_idNãostringID da sessão WhatsApp. Omita para usar a primeira conectada

Estrutura de itens

"itens": [ { "descricao": "Coca-Cola 2L", "quantidade": 2, "valor_unitario": "8,50" }, { "descricao": "Pão de Forma", "quantidade": 1, "valor_unitario": "12,00" } ]

Exemplo de requisição

POST http://localhost:4600/erp/venda
{ "phone": "44999990000", "numero_venda": "001234", "nome_cliente": "João Silva", "valor_total": "29,00", "forma_pagamento": "PIX", "data": "12/05/2026", "itens": [ { "descricao": "Coca-Cola 2L", "quantidade": 2, "valor_unitario": "8,50" }, { "descricao": "Pão de Forma", "quantidade": 1, "valor_unitario": "12,00" } ] }

Resposta de sucesso

{ "ok": true, "phone": "5544999990000", "sessao_id": "a1b2c3", "enviados": ["texto"] }

9. Endpoint Genérico (Qualquer Evento)

Use quando quiser aproveitar templates customizados ou enviar outros tipos de notificação.

POST /erp/notificar Evento + variáveis → template → WhatsApp
CampoReq.TipoDescrição
phoneSimstringNúmero destino
eventoSimstringNome do template. Ex: "venda_realizada"
varsNãoobjectVariáveis para preencher o template
arquivo_base64NãostringArquivo em base64
arquivo_nomeNãostringNome do arquivo com extensão
arquivo_captionNãostringLegenda do arquivo
apenas_arquivoNãobooltrue = não envia texto, só o arquivo
Exemplo com evento personalizado
{ "phone": "44999990000", "evento": "custom", "vars": { "mensagem": "Olá! Seu pedido já saiu para entrega. Previsão: 30 min. 🚚" } }
10. Templates de Mensagem

O PDV usa templates editáveis para montar as mensagens. As variáveis entre {chaves} são substituídas pelos dados enviados pelo ERP.

EventoDescriçãoVariáveis principais
venda_realizadaComprovante de vendanome_cliente numero_venda valor_total forma_pagamento data itens
nota_fiscalNF-e emitidanome_cliente numero_nf valor_total data chave_acesso link_danfe
boletoBoleto bancárionome_cliente valor_total data_vencimento linha_digitavel link_boleto
pedido_confirmadoConfirmação de pedidonome_cliente numero_pedido valor_total previsao_entrega
entrega_realizadaEntrega concluídanome_cliente numero_pedido data
cobrancaLembrete de pagamentonome_cliente valor_total data_vencimento
customMensagem livremensagem (texto completo)
💡 Editar templates: abra http://localhost:4600/erp/templates/painel no navegador da máquina do PDV para editar as mensagens visualmente. Também é possível via API: PUT /erp/templates.
✅ Variável não enviada? O marcador é mantido na mensagem (ex: se não enviar forma_pagamento, aparece {forma_pagamento}). Edite o template para remover campos opcionais que não usa.
11. Conectar o WhatsApp (QR Code)

Antes de usar o PDV, o WhatsApp precisa ser conectado escaneando um QR code. Este processo é feito uma única vez (ou quando o WhatsApp desconectar).

PassoAção
1Abra http://localhost:4600/setup/qr no navegador da máquina do PDV
2Uma página com QR code de configuração é exibida
3Para conectar o WhatsApp, acesse http://localhost:4600/whatsapp/sessoes e crie uma sessão
4Abra a página de QR da sessão e escaneie com o celular no WhatsApp
5Após conectar, o PDV já está pronto para enviar mensagens
⚠️ O WhatsApp pode desconectar se o celular ficar sem bateria ou sem internet. Oriente o operador a verificar o ícone do ZapDin PDV na bandeja do sistema. Clique no ícone → "Conectar WhatsApp" para reconectar.
12. Exemplos de Código
Python
import requests PDV_URL = "http://localhost:4600" HEADERS = {"X-PDV-Key": "minha-chave-secreta-123", "Content-Type": "application/json"} # Verificar status antes de enviar status = requests.get(f"{PDV_URL}/status", headers=HEADERS).json() if status["sessoes_conectadas"] == 0: print("WhatsApp não conectado!") else: # Enviar comprovante de venda r = requests.post(f"{PDV_URL}/erp/venda", headers=HEADERS, json={ "phone": "44999990000", "numero_venda": "001234", "nome_cliente": "João Silva", "valor_total": "350,00", "forma_pagamento": "PIX", "data": "12/05/2026", }) print(r.json()) # {"ok": true, ...}
C# / .NET
using System.Net.Http; using System.Text; using System.Text.Json; var client = new HttpClient(); client.DefaultRequestHeaders.Add("X-PDV-Key", "minha-chave-secreta-123"); var payload = new { phone = "44999990000", numero_venda = "001234", nome_cliente = "João Silva", valor_total = "350,00", forma_pagamento = "PIX", data = "12/05/2026" }; var json = JsonSerializer.Serialize(payload); var content = new StringContent(json, Encoding.UTF8, "application/json"); var resp = await client.PostAsync("http://localhost:4600/erp/venda", content); var result = await resp.Content.ReadAsStringAsync(); // result = {"ok":true,"phone":"5544999990000","enviados":["texto"]}

13. Tabela de Erros
HTTPSituaçãoComo resolver
401 Header X-PDV-Key ausente ou incorreto Verifique o valor de PDV_API_KEY no .env do PDV
400 Template não encontrado ou inativo Verifique o nome do evento. Consulte os ativos em GET /erp/templates
400 Erro ao enviar pelo WhatsApp (número inválido, bloqueado, etc.) Verifique o número do destinatário. Confira se o WhatsApp está conectado
503 Nenhuma sessão WhatsApp conectada Oriente o operador a reconectar o WhatsApp via QR code
422 Campo obrigatório ausente ou tipo inválido Verifique os campos marcados como Sim na tabela do endpoint
000 / timeout PDV não está rodando Verifique se o ícone do ZapDin PDV aparece na bandeja do sistema Windows
