1. Visão Geral

O ZapDin é um sistema instalado localmente no estabelecimento do cliente que automatiza o envio de mensagens e arquivos via WhatsApp. O ERP se comunica com ele por uma API REST local, sem depender de serviços externos pagos por mensagem.

Base URL O endereço base é fornecido pelo administrador do ZapDin. Normalmente é o IP local da máquina onde o ZapDin está instalado seguido da porta 4000.

Exemplo: http://192.168.1.100:4000

Todos os endpoints de integração com o ERP têm o prefixo /api/erp/.
Formato do telefone Envie apenas DDD + número, sem o DDI 55. O sistema adiciona o DDI automaticamente.
Correto: "11999990000"  →  o sistema usa "5511999990000"
Errado: "5511999990000", "+5511999990000", "(11) 99999-0000"

2. Como Funciona (Fluxo)

O ZapDin opera com uma fila assíncrona. Quando o ERP chama a API, a mensagem ou arquivo é imediatamente enfileirado e a API retorna {"ok": true, "queued": true} em milissegundos. Um processo worker em segundo plano processa a fila e efetua o envio, aplicando os delays anti-ban automaticamente.

Alta disponibilidade: se o WhatsApp estiver temporariamente desconectado, as mensagens ficam na fila e são enviadas automaticamente quando a conexão for restabelecida.

3. Autenticação

Toda requisição à API deve incluir o token de autenticação no header HTTP x-token. O token é gerado pelo administrador do ZapDin em Painel → Integrações → Token API.

Header HTTPValorObrigatório
x-token Token fornecido pelo administrador do ZapDin Sim
Content-Type application/json Sim 
# Headers obrigatórios em todas as requisições
x-token: rYq1OWgra44Nx_nlaSWF-VMBzObpfCMJY-5EZx9-MaA
Content-Type: application/json
Segurança: nunca inclua o token em código-fonte público (repositórios, documentos compartilhados). Armazene-o em variável de ambiente ou configuração criptografada no ERP. Se comprometido, o administrador pode gerar um novo token no painel — o token anterior é invalidado imediatamente.

4. Endpoint: Envio de Mensagem

POST /api/erp/venda Enfileira mensagem WhatsApp com dados da venda/atendimento

Recebe os dados de uma venda ou atendimento e enfileira automaticamente uma mensagem WhatsApp ao cliente. O conteúdo da mensagem é gerado a partir do template configurado no painel, substituindo as variáveis pelos dados enviados. Pode-se também enviar uma mensagem completamente personalizada usando o campo mensagem_custom.

Campos do body (JSON)

CampoTipoObrigatórioDescrição
telefone string Sim DDD + número. Sem o DDI 55 — inserido automaticamente.
Aceita: "11999990000", "1133334444"
Não aceita: "5511...", "+55...", "(11)..."
nome string Sim Nome completo do cliente. Usado na variável {nome} do template.
produtos array Não Lista de produtos/serviços consumidos.
Cada item: {"nome": "...", "quantidade": 1, "valor_unitario": "5,99"}
Usado na variável {produtos} do template.
valor_total_itens string Não Soma dos valores dos itens antes de descontos/taxas.
Exemplo: "281,60". Usado em {valor_total_itens}.
valor_total string Não Valor final da compra (com descontos e taxas aplicados).
Exemplo: "275,00". Usado em {valor_total}.
data string Não Data da venda no formato DD/MM/AAAA.
Se omitida, usa automaticamente a data atual do servidor.
mensagem_custom string Não Mensagem livre que substitui completamente o template configurado.
Use quando precisar de um texto totalmente diferente para casos específicos.

Estrutura de cada item em produtos

CampoTipoObrigatórioDescrição
nome string Sim Nome do produto ou serviço
quantidade number Não Quantidade consumida. Padrão: 1. Aceita decimais (ex: 40.5).
valor_unitario string Não Valor por unidade formatado como string. Exemplo: "5,99"

5. Endpoint: Envio de Arquivo

POST /api/erp/arquivo Enfileira envio de arquivo (PDF, imagem, etc.) via WhatsApp

Envia um arquivo ao cliente via WhatsApp — tipicamente uma nota fiscal eletrônica em PDF, comprovante ou imagem. O conteúdo do arquivo deve ser enviado codificado em Base64 dentro do JSON. O ZapDin salva o arquivo localmente e o envia através do WhatsApp.

Formatos suportados: qualquer formato que o WhatsApp aceite como documento ou imagem. Os mais comuns: .pdf, .jpg, .png, .xlsx, .docx. Para notas fiscais, use sempre .pdf.

Campos do body (JSON)

CampoTipoObrigatórioDescrição
telefone string Sim DDD + número. Sem o DDI 55 — inserido automaticamente.
nome_arquivo string Sim Nome do arquivo com extensão exibido no WhatsApp.
Exemplo: "nota_fiscal_001.pdf", "comprovante_25042026.pdf"
conteudo_base64 string Sim Conteúdo binário do arquivo codificado em Base64.
Sem o prefixo data:application/pdf;base64, — apenas os bytes codificados.
Ver exemplos de codificação abaixo.
mensagem string Não Legenda exibida junto ao arquivo no WhatsApp.
Exemplo: "Segue sua nota fiscal eletrônica. Obrigado pela preferência!"

Exemplo de requisição

POST http://192.168.1.100:4000/api/erp/arquivo
x-token: SEU_TOKEN_AQUI
Content-Type: application/json

{
  "telefone":        "11999990000",
  "nome_arquivo":    "nota_fiscal_00142.pdf",
  "conteudo_base64": "JVBERi0xLjQKJcOkw7zDtsOfCjIgMCBvYmoKPDwKL0xlbmd...",
  "mensagem":        "Olá João! Segue sua nota fiscal eletrônica. Obrigado!"
}
200 OK Resposta de sucesso 
{
  "ok":     true,
  "queued": true
}

6. Template de Mensagem

O texto enviado ao cliente é gerado a partir de um template configurável pelo administrador do ZapDin em Painel → Configurar Mensagem. O template pode usar variáveis entre chaves {variavel} que são substituídas automaticamente pelos dados enviados pelo ERP.

Variável no templateCampo enviado pelo ERPExemplo de saída
{nome} nome João Silva
{telefone} telefone (já com DDI 55) 5511999990000
{data} data ou data atual 25/04/2026
{produtos} Lista formatada do array produtos • Gasolina Comum (x40) — R$ 5,99
• Óleo Motor 5W30 (x1) — R$ 42,00
{valor_total_itens} valor_total_itens 281,60
{valor_total} valor_total 275,00

✅ Total:    R$ {valor_total}

Mensagem customizada: se o campo mensagem_custom for enviado, ele substitui completamente o template — as variáveis acima são ignoradas. Use para casos especiais como alertas, lembretes ou mensagens com formato totalmente diferente.

7. Códigos de Resposta HTTP

CódigoSituaçãoBody de respostaAção recomendada
200 OK Mensagem/arquivo aceito e enfileirado {"ok": true, "queued": true} Registrar como enviado no ERP
401 Unauthorized Token ausente ou inválido {"detail": "Token inválido"} Verificar o token configurado no ERP
422 Unprocessable Entity Campo obrigatório ausente ou tipo errado {"detail": [...]} Corrigir o payload — verificar campos obrigatórios
503 Service Unavailable Nenhuma sessão WhatsApp conectada {"detail": "..."} Aguardar; notificar administrador para reconectar o WhatsApp
500 Internal Server Error Erro interno no servidor ZapDin {"detail": "..."} Aguardar e tentar novamente; verificar logs no painel
Timeout / Sem resposta ZapDin offline ou rede indisponível — Verificar se o serviço ZapDin está em execução; aguardar e tentar novamente

8. Tratamento de Erros

Recomenda-se implementar a lógica abaixo no ERP para garantir que nenhuma mensagem se perca em caso de falha temporária:

CódigoDeve tentar novamente?Estratégia
200 Não Sucesso — registrar envio
401 Não Erro de configuração — corrigir o token, não retentar automaticamente
422 Não Erro no payload — corrigir os dados enviados
503 Sim Aguardar 30–60 segundos e tentar novamente (máx. 3 tentativas)
500 Sim Aguardar 30 segundos e tentar novamente (máx. 3 tentativas)
Timeout / Conexão recusada Sim Salvar em fila local do ERP e tentar novamente quando o serviço voltar
Timeout recomendado: configure o timeout da requisição HTTP no ERP para 10 segundos. O ZapDin responde em milissegundos (apenas enfileira), portanto qualquer demora indica problema de rede.

// ── Função auxiliar para POST JSON ───────────────────────────────────
function ZapDinPost(const Endpoint, JsonBody: string): Integer;
var
  Http    : TNetHTTPClient;
  Stream  : TStringStream;
  Resp    : IHTTPResponse;
begin
  Http   := TNetHTTPClient.Create(nil);
  Stream := TStringStream.Create(JsonBody, TEncoding.UTF8);
  try
    Http.ConnectionTimeout := 10000;
    Http.ResponseTimeout   := 10000;
    Http.CustomHeaders['x-token']      := TOKEN;
    Http.CustomHeaders['Content-Type'] := 'application/json';
    Resp   := Http.Post(BASE_URL + Endpoint, Stream);
    Result := Resp.StatusCode;
  finally
    Stream.Free;
    Http.Free;
  end;
end;

// Java — usando java.net.http (Java 11+)
import java.net.http.*;
import java.net.URI;
import java.nio.file.Files;
import java.nio.file.Paths;
import java.time.Duration;
import java.util.Base64;

public class ZapDinClient {

    private static final String BASE_URL = "http://192.168.1.100:4000";
    private static final String TOKEN    = "SEU_TOKEN";

    private final HttpClient http = HttpClient.newBuilder()
        .connectTimeout(Duration.ofSeconds(10))
        .build();

    // ── Enviar mensagem de venda ──────────────────────────────────────
    public void enviarVenda(String telefone, String nomeCliente,
                             String valorTotal) throws Exception {
        String json = String.format("""
            {
              "telefone": "%s",
              "nome":     "%s",
              "valor_total": "%s"
            }""", telefone, nomeCliente, valorTotal);

        var req = HttpRequest.newBuilder()
            .uri(URI.create(BASE_URL + "/api/erp/venda"))
            .timeout(Duration.ofSeconds(10))
            .header("x-token", TOKEN)
            .header("Content-Type", "application/json")
            .POST(HttpRequest.BodyPublishers.ofString(json))
            .build();

        var resp = http.send(req, HttpResponse.BodyHandlers.ofString());
        if (resp.statusCode() != 200)
            throw new RuntimeException("ZapDin erro: " + resp.statusCode());
    }

    // ── Enviar arquivo PDF ────────────────────────────────────────────
    public void enviarArquivo(String telefone, String caminhoArquivo,
                               String nomeExibicao) throws Exception {
        byte[] bytes  = Files.readAllBytes(Paths.get(caminhoArquivo));
        String b64    = Base64.getEncoder().encodeToString(bytes);
        String json   = String.format("""
            {
              "telefone":        "%s",
              "nome_arquivo":    "%s",
              "conteudo_base64": "%s",
              "mensagem":        "Segue sua nota fiscal eletrônica!"
            }""", telefone, nomeExibicao, b64);

        var req = HttpRequest.newBuilder()
            .uri(URI.create(BASE_URL + "/api/erp/arquivo"))
            .timeout(Duration.ofSeconds(15))
            .header("x-token", TOKEN)
            .header("Content-Type", "application/json")
            .POST(HttpRequest.BodyPublishers.ofString(json))
            .build();

        http.send(req, HttpResponse.BodyHandlers.ofString());
    }
}

// PHP — usando cURL
define('ZAPDIN_URL', 'http://192.168.1.100:4000');
define('ZAPDIN_TOKEN', 'SEU_TOKEN');

function zapdin_post($endpoint, $dados): array {
    $ch = curl_init(ZAPDIN_URL . $endpoint);
    curl_setopt_array($ch, [
        CURLOPT_POST           => true,
        CURLOPT_POSTFIELDS     => json_encode($dados, JSON_UNESCAPED_UNICODE),
        CURLOPT_HTTPHEADER     => [
            'x-token: ' . ZAPDIN_TOKEN,
            'Content-Type: application/json',
        ],
        CURLOPT_RETURNTRANSFER => true,
        CURLOPT_TIMEOUT        => 10,
    ]);
    $body    = curl_exec($ch);
    $status  = curl_getinfo($ch, CURLINFO_HTTP_CODE);
    curl_close($ch);
    if ($status !== 200)
        throw new RuntimeException("ZapDin retornou HTTP {$status}");
    return json_decode($body, true);
}

10. Boas Práticas

TópicoRecomendação
Timeout Configure 10 segundos para o endpoint /venda e 30 segundos para /arquivo (arquivos grandes levam mais tempo para codificar/transmitir).
Retry Para erros 500 e 503, tente novamente até 3 vezes com intervalo de 30 segundos. Para timeout/conexão recusada, salve em fila local e tente quando o serviço retornar. Nunca retente erros 401 e 422 automaticamente.
Fila local Implemente uma fila simples no ERP (banco de dados) para persistir mensagens pendentes em caso de indisponibilidade do ZapDin. Processe a fila periodicamente.
Tamanho de arquivo Limite arquivos a 16 MB (limite do WhatsApp para documentos). Para PDFs de NF-e, o tamanho típico é de 50–200 KB, bem abaixo do limite.
Formato do telefone Sempre remova caracteres não numéricos antes de enviar. Envie apenas DDD + número, sem espaços, parênteses, hífens ou DDI. Celulares têm 11 dígitos (DDD + 9 dígitos), fixos têm 10 (DDD + 8 dígitos).
Horário de envio O ZapDin enfileira imediatamente, mas considere chamar a API apenas em horário comercial para não enviar mensagens à madrugada. Isso é controlado pelo ERP, não pelo ZapDin.
Log de auditoria Registre no ERP: timestamp da chamada, telefone, código de resposta HTTP e queued: true/false. Isso facilita o diagnóstico em caso de problemas.
Segurança do token Armazene o token em variável de ambiente ou tabela de configurações do ERP, nunca em código-fonte. Use HTTPS em produção se o ZapDin estiver em servidor remoto.
Checklist de integração
☐ Token configurado no ERP (x-token)
☐ Base URL apontando para o IP/porta correto do ZapDin
☐ Telefone sendo enviado sem DDI, sem formatação
☐ Timeout definido (10s para mensagens, 30s para arquivos)
☐ Tratamento de erro implementado (retry para 500/503)
☐ Log de auditoria no ERP
☐ WhatsApp conectado no painel do ZapDin antes de testar
